class evclient.client.EVClient(domain: Optional[str] = None, api_key: Optional[str] = None)

Bases: evclient.csv_import_client.CSVImportClient, evclient.node_client.NodeClient, evclient.tag_client.TagClient, evclient.settings_client.SettingsClient, evclient.timeseries_client.TimeseriesClient, evclient.dataset_client.DatasetClient

A class for handling all sections of NODA EnergyView API combined into one client

__init__(domain: Optional[str] = None, api_key: Optional[str] = None)

BaseClient constructor

Defines the base url and customer domain for the EnergyView API and sets headers in a new requests session.

Will look for the following environment variables if parameters are omitted:

EV_DOMAIN

EV_API_KEY

Parameters

  • domain (Optional[str]) – The EnergyView domain to make requests to.

  • api_key (Optional[str]) – API Key for the selected EnergyView domain.

Raises

EVFatalErrorException – The client could not find a specified domain or api key for the EnergyView API

get_settings(settings_type: str, settings_id: int, path: Optional[str] = None, extract: bool = False) → Dict

Fetches all settings / metadata of a resource from EnergyView API

Parameters

  • settings_type (str) –

    One of:

    • cluster

    • node

    • report

    • netmgr

  • settings_id (int) – The numeric id associated with the resource.

  • path (Optional[str]) – Filter using the specified path. Separator is ‘.’ (dot). For example: coco.default.

  • extract (Optional[bool]) –

    Unique identifier for the integration. When set to True, everything beneath path will be extracted and returned. For example:

    extract=False and path=coco.default
Result:
{"coco": {"default": {"hello": "world"}}}
Compared to:
extract=True and path=coco.default
Result:
{"hello": "world"}

Or:
extract=True and path=coco.default.hello
Result:
"world"

Returns

Dict of settings, keys will vary depending on the settings_type and resource requested

Raises

  • EVBadRequestException – Sent request had insufficient data or invalid options.

  • EVUnauthorizedException – Request was refused due to lacking authentication credentials.

  • EVForbiddenException – Server understands the request but refuses to authorize it.

  • EVNotFoundException – The requested resource was not found.

  • EVTooManyRequestsException – Sent too many requests in a given amount of time.

  • EVInternalServerException – Server encountered an unexpected condition that prevented it from fulfilling the request.

get_timeseries_data(node_id: Optional[int] = None, node_ids: Optional[List[int]] = None, tag: Optional[str] = None, tags: Optional[List[str]] = None, start: Optional[str] = None, end: Optional[str] = None, resolution: Optional[str] = None, aggregate: Optional[str] = None, epoch: bool = False) → evclient.types.timeseries_types.TimeseriesResponse

Fetches all timeseries data from EnergyView API

Parameters

  • node_id (Optional[int]) – Filter on the unique identifier for a specific node.

  • node_ids (Optional[List[int]]) – Filter on several unique node identifiers. Can not be used together with node_id.

  • tag (Optional[str]) – Filter on a sensor name.

  • tags (Optional[List[str]]) – Filter on several sensor names. Can not be used together with tag.

  • start (Optional[str]) – The from date time string on the format YYYY-MM-DDThh:mm:ss±hh:mm. Without timezone information, the API will fall back to the time zone configured for the domain. Defaults to now.

  • end (Optional[str]) – The from date time string on the format YYYY-MM-DDThh:mm:ss±hh:mm. Without timezone information, the API will fall back to the time zone configured for the domain. Defaults to now.

  • resolution (Optional[str]) –

    Truncates all timestamps to any of (options), then computes the average for all points within the truncated period.

    • second

    • minute

    • 5minute

    • 10minute

    • 15minute

    • 20minute

    • 30minute

    • hour

    • day

    • month

    • year

    • decade

    • century

    • millennia

  • aggregate (Optional[str]) –

    When using resolution, select this aggregate function instead of the default avg when computing the result.

    • avg

    • min

    • max

    • sum

    • count

  • epoch (Optional[bool]) – When set to True, the ts field will be in Unix timestamp format (numeric) instead of a string. This is the number of seconds that have elapsed since the Unix epoch, which is the time 00:00:00 UTC on 1 January 1970.

Returns

TimeseriesResponse

Raises

  • EVBadRequestException – Sent request had insufficient data or invalid options.

  • EVUnauthorizedException – Request was refused due to lacking authentication credentials.

  • EVForbiddenException – Server understands the request but refuses to authorize it.

  • EVTooManyRequestsException – Sent too many requests in a given amount of time.

  • EVInternalServerException – Server encountered an unexpected condition that prevented it from fulfilling the request.

store_multiple_timeseries_data(timeseries: List[evclient.types.timeseries_types.TimeseriesType], overwrite: Optional[str] = None, silent: Optional[bool] = None) → Optional[evclient.types.timeseries_types.TimeseriesResponse]

Store multiple data points in multiple timeseries from EnergyView API

Parameters

  • timeseries (List[TimeseriesType]) –

    A list of timeseries objects:

    [
    {
        'node_id': 1,
        'tag': 'outdoortemp',
        'data': [{
            'v': 2.6,
            'ts': '2020-01-01T00:00:00'
        },{
            'v': 2.6,
            'ts': '2020-01-01T00:15:00'
        }]
    }
]

  • overwrite (Optional[str]) –

    Possible values are:

    • replace_window

    replace_window first deletes all datapoints between the lowest and highest ts (a >= x AND a <= y), for each node_id and corresponding tag. Then inserts all the new datapoints.

  • silent (Optional[bool]) – When set to true a call will only reply with status code 201 Created and an empty reply instead of 200 Success and the inserted rows.

Returns

TimeseriesResponse or None depending on if the silent param is set.

Raises

  • EVUnauthorizedException – Request was refused due to lacking authentication credentials.

  • EVForbiddenException – Server understands the request but refuses to authorize it.

  • EVTooManyRequestsException – Sent too many requests in a given amount of time.
